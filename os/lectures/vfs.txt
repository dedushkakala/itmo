	- Джеф Дин - какие числа должен знать каждый программист
		- один цикл - 10^(-9) секунд
		- обращение к памяти - 100-200 тактов
	- выводы
		- читать нужно последовательно
	- файл
		- абстракция для данных и обычно как читать делает пользователь
		- все представимо ввиде файлов
		- у каждого файла есть модификаторы доступа(read write execute) и есть 3 группы - пользователь группа и все остальные. Поэтому есть у каждого файла 9 букв - rw-rw-r-- например
		
	-директории
		- директори - способ организовать файлы. ls - посмотреть все файлы.
		- во всех есть . - текушая директория и .. - родитель.
		- r - получить список файлов, x - возможность обращения к файлу в директории, w - изменение содержимой директории.

	-suid & sgid
		- suid бит - сделать так, чтобы при запуске привелегии были владельца этого файла(например в sudo Он стоит и там пользователь root - то есть запустить прогу от root)
		- sticky bit - флаг для того, чтобы изменить можно было только руту или создателю и группе
	-inode
		- Штука которая хранит служебную информацию о файле
		- это как контрол блок в shared_ptr
		- бывают легкие ссылки и тяжелые ссылки. Тяжелые - два файла с одинаковым содержимым. Легкие - тип ярлыка в винде. Тяжелые - только в своей системе, легкие - могут за пределы системы.
	-иерархия
		- mount - команда, чтобы подсоединить файловую систему как поддерево
		- в unix в одном дереве могут быть разные файловые системы. Могут быть даже по интернету
		- чтобы ходить по дерево есть chdir - системный вызов. И по факту cd - вызов этого системного вызова
		- mkdir - сист вызов чтобы создать директорию
	-файловые дескримторы
		- creat - создается новый файловый дескриптор(open c O_CREATE)
		- stdin - 0 stdout - 1 stderr -2
		- при создании создается наименьший текущий свободный номер 
		- getdents - чтобы получить список файлов в директории
		- read и write для того, чтобы прочитать и записать
		- lseak - системный вызов(как fseak)
		- close - закрыть
		- TOCTOU - time of check to time of use - проблема с атомарностью.
