Структура оси
                                                системные библиотеки(libc, libdl, libsocket)
    virtualization                               syscalls(не библиотечная функция, но для прогера это одно и тоже)
         |                                          |
Hardware<->Kernel space(общаяется с ядром напрямую)<->User space(наши програмки - Chrome)
                |
            "Монолитный бинарь"
Системные библиотеки и user space хранится в ABI(application binary interface).

Из чего состоит ядро
1)BOOT(как загружается какие процессы протекают)
2)VFS(Virtual File System - подсистема которая отвечает за файловую систему)
3)VM(Virtual Memory - организация виртуальной памяти)
4)NET(работа с сетью)
5)DRIVERS(Драйвера для железа)
6)SCHEDULER(обработка прерываний, думает что сейчас выполнять)
7)VIRT(virtualization)
8)ARCH(специфичный для архитектуры код)
9)SYSCALL(как из юзер спейса мы общаемся с ядром)

Из чего состоит юзерспейс(это все общается через syscalls)
1)BOOST/QT(библиотеки)
2)libc(memory allocator)
3)resolving(dns)
4)NSS(network service switching служебные сервисы)

Кроме этого
.)IPS(Inter-process communication - взаимодействие между процессами( например в хроме есть отдельные процессы которые рабтоают с одним блокомц))
